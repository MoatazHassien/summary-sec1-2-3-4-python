Python Syntax - شكل الكود

إزاي بايثون بيفهم كودك

أول حاجة لازم تفهمها في بايثون هي إن المسافات (Indentation) مش تنسيق - دي جزء من اللغة نفسها!

في لغات زي Java أو C++، بتستخدم {} أقواس عشان تقول "الكود ده جوه الـ if مثلاً"... في بايثون مفيش أقواس، بتعمل مسافة (4 مسافات أو TAB) وده اللي بيقول للبايثون "الكود ده تبع السطر اللي فوقه".

تخيل إنك بتكتب رسالة وعايز تقول "ده فقرة فرعية"... بتعمل indent يعني تدخل شوية من الشمال. بايثون بيعمل نفس الفكرة بس إجباري مش اختياري!

بايثون مش بيحتاج سيمي كولون ; في آخر كل سطر زي Java. كل سطر هو سطر منفصل أوتوماتيك.
لو نسيت المسافة أو عملت مسافتين بدل أربعة، الكود هيديك error اسمه IndentationError - ده اكتر غلطة ممكن تغلطها
python
# سطر عادي - من غير مسافة x = 1 print(x) # هنا الـ if محتاج مسافة للكود اللي جواه if 5 > 2: print("خمسة أكبر من اتنين!") # ← 4 مسافات # مفيش ; في آخر السطر زي C++ أو Java name = "FCDS" print(name)
OUTPUT
1 خمسة أكبر من اتنين! FCDS
02

Output - إظهار الكلام على الشاشة

كل حاجة تخص الfunction print()

الـ print() دي الأهم لأنها بتخليك تشوف نتيجة كودك على الشاشة. بس فيها تفاصيل كتير !

الـ print العادية بتطبع الكلام وفي الآخر بتعمل سطر جديد أوتوماتيك. ممكن تغير ده بالـ end.

لو عايز تطبع أكتر من حاجة في نفس الأمر، بتفصلهم بفاصلة، وأوتوماتيك هيحط مسافة بينهم. ممكن تغير المسافة دي بالـ sep.

python
# طباعة عادية print("Material Team") # طباعة أكتر من حاجة - بيحط مسافة بينهم تلقائياً print("بايثون", "سهلة", "اوي!") # sep: تغيير الفاصل بين الكلام print("بايثون", "سهلة", "اوي!", sep="-") # end: بدل ما يعمل سطر جديد، هيحط مسافة print("Hello", end=" ") print("QUIZ")
OUTPUT
Material Team بايثون سهلة اوي! بايثون-سهلة-اوي! Hello QUIZ
لاحظ إن السطر الأخير طلع على نفس السطر عشان استخدمنا end=" " وده خلى print الأولى ما تعملش سطر جديد وبدله حطت مسافة، فالـ print التانية كملت على نفس السطر.

Comments - التعليقات

الكلام اللي بايثون بيتجاهله تماماً

التعليقات هي كلام بتكتبه في الكود بايثون بيتجاهله تماماً ومش بيشتغله. فايدتها إيه؟

  • تشرح الكود لنفسك أو لزمايلك
  • تلغي سطر كود مؤقتاً من غير ما تمسحه
  • توثق الكود بتاعك

التعليق بسطر واحد: تبدأ بـ #

التعليق بأكتر من سطر بس دي الزيرو قال مش صح اوي: تحطه جوه """ تلاتة كوتيشن أو '''

في الكورس نفسه، التعليقات بتشرح الكود - اتعودوا تكتبوا تعليقات دايماً لأن الكود من غير تعليقات بيبقى صعب تفهمه بعد فترة حتى لو إنت اللي كتبته!
python
# ده تعليق بسطر واحد - بايثون هيتجاهله # ممكن تلغي سطر مؤقتاً كده: # print("السطر ده موقوف مؤقتاً") x = 5 # تعليق على نفس السطر مع الكود """ ده تعليق على أكتر من سطر بنستخدمه لشرح حاجة طويلة بس دي الزيرو قال مش صح اوي """ ''' ممكن نفس الفكرة باستخدام single quotes ''' print("الكود بيشتغل عادي!")
04

Variables - المتغيرات

صناديق بنحفظ فيها البيانات

المتغير هو اسم بتديه لمكان في الذاكرة بتحفظ فيه قيمة. تخيل إنك عندك صناديق وكل صندوق ليه اسم، تقدر تحط فيه أي حاجة وتاخد منه أي وقت.

تخيل إنك في مطبخ وعندك علب. كتبت على علبة "ملح"، وعلى تانية "سكر"، وعلى تالتة "عمر". في أي وقت تجي تطبخ، تقدر تاخد من العلب دي باسمها. المتغيرات نفس الفكرة!

في بايثون مش محتاج تقول للبرنامج "المتغير ده هيكون رقم" أو "هيكون نص" - بايثون بيعرف لوحده من القيمة اللي حطيتها. ده اللي بيسموه Dynamically Typed.

قواعد تسمية المتغيرات: لازم يبدأ بحرف أو underscore _ - ما ينفعش يبدأ برقم. ممكن يحتوي على حروف وأرقام وunderscore. case-sensitive يعني age و Age و AGE تلاتة متغيرات مختلفين! ما ينفعش يكون كلمة محجوزة في بايثون زي if أو for.
python
# إنشاء متغيرات name = "Moataz" #نص age = 19 # رقم صحيح height = 1.75 # رقم عشري is_engineer = True # صح أو غلط print(name) print(age) print(height) print(is_engineer) # إسناد أكتر من متغير في سطر واحد! x, y, z = 1, 2, 3 print(x, y, z) # إعطاء نفس القيمة لأكتر من متغير a = b = c = 10 print(a, b, c)
OUTPUT
Moataz 19 1.75 True 1 2 3 10 10 10

البيانات في بايثون بتيجي بأشكال مختلفة. عشان تعرف إيه نوع المتغير بتاعك، بتستخدم type().

النوع الاسم بالإنجليزي مثال الاستخدام
رقم صحيح int x = 10 أعداد كاملة بدون كسور
رقم عشري float x = 3.14 أعداد بكسور عشرية
نص str x = "مرحبا" أي نص أو حروف
صح/غلط bool x = True قيم منطقية
قائمة list x = [1, 2, 3] مجموعة قابلة للتعديل

name = "Moataz" age = 19 height = 175.8 grades = [90, 85, 92] info = {"city": "القاهرة"} print(type(name)) # <class 'str'> print(type(age)) # <class 'int'> print(type(height)) # <class 'float'> print(type(grades)) # <class 'list'> print(type(info)) # <class 'dict'>

Numbers - الأرقام

int و float و complex وعمليات الحساب

في بايثون في تلات أنواع أرقام:

  • int: أرقام صحيحة زي 5, -3, 100, 0
  • float: أرقام عشرية زي 3.14, -2.5, 1.0
  • complex: أرقام مركبة (هندسة كهربية ورياضيات) زي 3+5j

العمليات الحسابية الأساسية:

الرمز العملية مثال النتيجة
+جمع10 + 313
-طرح10 - 37
*ضرب10 * 330
/قسمة (بكسور)10 / 33.333...
//قسمة صحيحة (تقريب للأسفل)10 // 33
%باقي القسمة (Modulus)10 % 31
**أس (قوة)2 ** 38

الـ % (Modulus) مهمة جداً! مثلاً 10 % 3 = 1 يعني إيه؟ يعني 10 ÷ 3 = 3 وباقي 1. استخدامها الشهير: عايز تعرف الرقم زوجي ولا فردي؟ x % 2 == 0 ده زوجي!

07

Casting - تحويل الأنواع

بدّل من نوع لنوع تاني

أحياناً بيكون عندك بيانات من نوع معين وعايز تحولها لنوع تاني. مثلاً عندك الرقم 5 كنص "5" وعايز تجمعه مع رقم - محتاج تحوله لـ int الأول.

تخيل إن عندك 50 جنيه ورقي (نص) وعايز تصرفهم في الكانتين (عمليات حساب). لازم أول ما تحولهم لفلوس حقيقية (int/float). الـ Casting زي الصراف!

🔢 int()

بتحول لرقم صحيح. لو حولت float لـ int، بيشيل الكسر من غير تقريب. int(5.9) = 5

🔣 float()

بتحول لرقم عشري. float(5) = 5.0 و float("3.14") = 3.14

📝 str()

بتحول لنص. str(100) = "100" مفيدة لما تعايز تجمع نص مع رقم.

✅ bool()

بتحول لـ True/False. الصفر والفاضي = False. أي قيمة تانية = True.

python
x = int(5.9) # 5 (مش 6 - بيشيل الكسر مش بيقرب!) y = int("10") # 10 من نص a = float(5) # 5.0 m = str(100) # "100" - رقم لنص # bool: قيم False print(bool(0)) # False print(bool("")) # False - نص فاضي print(bool(None)) # False # bool: قيم True print(bool(1)) # True print(bool("معتز")) # True - نص فيه حاجة print(bool([1,2])) # True - قائمة فيها عناصر

Strings - النصوص

كل حاجة عن التعامل مع الكلام

الـ String هو أي نص أو حروف بتحطه جوه كوتيشن - سواء single 'hello' أو double "hello" أو triple """hello""" للنصوص على أكتر من سطر.

تخيل الـ string كأنه قائمة من الحروف. كل حرف ليه رقم (index) بيبدأ من 0. يعني في كلمة "Python":

الحرف Python
الـ Index (+) 012345
الـ Index (-) -6-5-4-3-2-1
python
text = "Python" # الوصول لحرف معين print(text[0]) # P - أول حرف print(text[2]) # t - الحرف التالت print(text[-1]) # n - آخر حرف # Slicing: جزء من النص [start:end] # الـ end مش داخل - يعني text[0:3] = 0,1,2 مش 3 print(text[0:3]) # Pyt print(text[2:]) # thon (لحد الآخر) print(text[:4]) # Pyth (من الأول) # String Methods - أساليب التعامل مع النص msg = " hello world " print(msg.upper()) # HELLO WORLD - كل حروف كبيرة print(msg.lower()) # hello world - كل حروف صغيرة print(msg.strip()) # hello world - شيل المسافات من الأطراف print(msg.replace("world", "Python")) # استبدال print(msg.split()) # ['hello', 'world'] - تقسيم لقائمة # طول النص print(len("Python")) # 6
الـ Methods على النص مش بتغير النص الأصلي - بتديك نص جديد. يعني لو كتبت msg.upper() ومتحفظتيش النتيجة في متغير، النص الأصلي فضل زي ما هو!

Booleans - صح وغلط

True و False وعمليات المقارنة

الـ Boolean نوع بيانات بياخد قيمتين بس: True (صح) أو False (غلط). لاحظ الحرف الأول كبير في بايثون!

بتيجي من عمليات المقارنة زي: هل 10 أكبر من 5؟ True. هل 3 يساوي 7؟ False.

العمليات المنطقية (and, or, not) بتدمج أكتر من شرط:

  • and: الاتنين لازم يكونوا True عشان النتيجة True
  • or: واحد بس كافي يكون True
  • not: بتعكس القيمة - True بيبقى False والعكس
python
print(10 > 5) # True print(10 < 5) # False print(10 == 10) # True - تساوي بنكتبه == print(10 != 5) # True - مش يساوي print(5 >= 5) # True - أكبر من أو يساوي print(5 <= 4) # False # العمليات المنطقية print(True and True) # True print(True and False) # False - واحد False يخلي الكل False print(True or False) # True - واحد True يكفي print(not True) # False - عكس

Operators - المعاملات

كل عمليات الحساب والمقارنة والإسناد

الـ Operators أنواع:

  • Arithmetic: +, -, *, /, //, %, ** عمليات الحساب
  • Assignment: =, +=, -=, *=, /= عمليات الإسناد المختصرة
  • Comparison: ==, !=, >, <, >=, <= مقارنة
  • Logical: and, or, not منطقية
الـ Assignment Operators المختصرة مهمة: بدل ما تكتب x = x + 3 تكتب x += 3 وبتعمل نفس الشغل!
python
x = 5 x += 3 # x = x + 3 = 8 print(x) # 8 x -= 2 # x = x - 2 = 6 x *= 2 # x = x * 2 = 12 print(x) # 12 # تمرين الكورس: (15 + 5) * 2 - 10 / 2 result = (15 + 5) * 2 - 10 / 2 print(result) # 35.0

Lists - القوائم

مجموعة عناصر في متغير واحد

الـ List هي متغير بتحفظ فيه أكتر من قيمة في نفس الوقت. بتعملها بـ [] أقواس مربعة والعناصر بينهم فاصلة.

تخيل إنك عندك قائمة تسوق: خبز، جبنة، لبن، بيض. ده قائمة! تقدر تضيف لها، تمسح منها، تغير فيها. الـ List في بايثون نفس الفكرة.

خواص الـ List:

  • Ordered: العناصر بترتيب ثابت وكل واحد ليه index
  • Changeable: تقدر تضيف وتمسح وتغير
  • Allows duplicates: نفس القيمة ممكن تتكرر
  • Any type: تقدر تحط أنواع مختلفة جوا نفس القائمة
python
fruits = ["تفاح", "موز", "كيوي"] # الوصول للعناصر print(fruits[0]) # تفاح - أول عنصر print(fruits[-1]) # كيوي - آخر عنصر # تعديل عنصر fruits[1] = "مانجو" print(fruits) # ['تفاح', 'مانجو', 'كيوي'] # إضافة عناصر fruits.append("برتقال") # إضافة للآخر fruits.insert(1, "عنب") # إضافة في موضع معين # حذف عناصر fruits.remove("كيوي") # احذف بالاسم last = fruits.pop() # احذف الأخير وخد قيمته # عمليات مفيدة numbers = [3, 1, 4, 1, 5] numbers.sort() # رتب تصاعدي numbers.reverse() # اعكس print(len(numbers)) # 5 - عدد العناصر print(numbers.count(1)) # كام مرة اتكررت 1

Tuples - السلاسل الثابتة

زي القائمة بس محدش يقدر يغير فيها

الـ Tuple زي الـ List بالظبط بس بتعملها بـ () أقواس عادية والأهم إنها Immutable يعني مقدرش أغير فيها بعد ما أعملها!

بص بقي الفرق بين ال tuple و ال list خلي بالك منه

إمتى تستخدم Tuple بدل List؟

  • لما البيانات ما المفروضش تتغير (زي إحداثيات جغرافية، تواريخ ثابتة)
  • أسرع من القائمة في التنفيذ
  • ممكن تستخدمها كـ key في القاموس (List ما تنفعش)
python
coords = (30.06, 31.24) # إحداثيات القاهرة fruits = ("تفاح", "موز", "كيوي") # الوصول زي القائمة print(fruits[0]) # تفاح print(fruits[-1]) # كيوي # Tuple Unpacking - رائعة! x, y = (10, 20) print(x) # 10 print(y) # 20 lat, lon = coords print(f"خط العرض: {lat}, خط الطول: {lon}") # محتاج تعد؟ count و index بس numbers = (1, 2, 2, 3, 2) print(numbers.count(2)) # 3 - كام مرة print(numbers.index(3)) # 3 - في الـ index انهي

Sets - المجموعات

عناصر فريدة بدون تكرار وبدون ترتيب

الـ Set مجموعة من العناصر بتعملها بـ {} أقواس معقوصة. خواصه:

  • بيشيل التكرار تلقائياً - لو حطيت نفس العنصر مرتين، هيحتفظ بنسخة واحدة بس
  • بدون ترتيب - ما فيش index، ما تقدرش تعمل set[0]
  • تقدر تضيف وتحذف - مش زي الـ Tuple

دي بقي في الوسط منه ولا زي ال list ولا زي ال tuple

python
# التكرار بييجي تلقائياً nums = {1, 2, 2, 3, 3, 3, 4} print(nums) # {1, 2, 3, 4} - اتشيل التكرار # إضافة وحذف fruits = {"تفاح", "موز", "كيوي"} fruits.add("برتقال") fruits.remove("موز") # التحقق من وجود عنصر print("تفاح" in fruits) # True # عمليات Sets - مفيدة جداً! set1 = {1, 2, 3, 4} set2 = {3, 4, 5, 6} print(set1 | set2) # Union: الكل {1,2,3,4,5,6} print(set1 & set2) # Intersection: المشترك {3,4} print(set1 - set2) # Difference: في الأول بس {1,2}

Dictionaries - القواميس

بيانات بمفتاح وقيمة - زي قاموس حقيقي

الـ Dictionary بتحفظ فيه البيانات كـ Key: Value يعني كل حاجة ليها اسم (مفتاح) وقيمة. بتعمله بـ {} وبتفصل بين الاسم والقيمة بـ :

تخيل القاموس الحقيقي: بتدور على كلمة (Key) وبتلاقي معناها (Value). قاموس بايثون نفس الفكرة: عندك "معتز": "name" - المفتاح "name" والقيمة "معتز".

القواميس مهمة جداً في البرمجة لأنها بتخلي البيانات منظمة ومفهومة.

python
person = { "name": "معتز", "age": 25, "city": "القاهرة" } # الوصول للقيم print(person["name"]) # معتز print(person.get("age")) # 25 (أأمن - ما بيديش error) # إضافة وتعديل person["job"] = "مهندس" # إضافة جديد person["age"] = 26 # تعديل موجود # حذف person.pop("city") # استعراض المحتوى print(person.keys()) # كل المفاتيح print(person.values()) # كل القيم print(person.items()) # كل الأزواج # التحقق من وجود مفتاح print("name" in person) # True print("email" in person) # False # قواميس متداخلة (Nested) students = { "student1": {"name": "علي", "grade": "A"}, "student2": {"name": "سارة", "grade": "B"} } print(students["student1"]["name"]) # علي

خلي الكود ياخد قرارات

While Loops - حلقة while

كرر طول ما الشرط صح

الـ while loop بتكرر كود معين طول ما شرط معين صح. لما الشرط يبقى غلط، الحلقة بتوقف.

خطر! لو الشرط فضل صح للأبد، الكود هيشتغل للأبد وده اللي بيسموه Infinite Loop ومهيفضلش يوقف إلا لما تقفله إنت بنفسك!

الـ while بييجي مع كلمتين مهمتين:

  • break: وقف الحلقة فوراً من غير ما تكمل
  • continue: تخطى باقي الكود في الدورة دي وروح للدورة الجاية
  • else: بيتنفذ بعد ما الحلقة خلصت بشكل طبيعي (من غير break)
python
# while بسيط count = 1 while count <= 5: print(count) count += 1 # مهم! لو نسيت ده = infinite loop # break: وقف مبكر i = 1 while i <= 10: if i == 5: break # وقف لما i يوصل لـ 5 print(i) i += 1 # هيطبع 1 2 3 4 بس # continue: تخطى دورة j = 0 while j < 5: j += 1 if j == 3: continue # تخطى الـ 3 print(j) # هيطبع 1 2 4 5 (بدون 3) # else مع while counter = 1 while counter <= 3: print(counter) counter += 1 else: print("الحلقة خلصت بنجاح!")

For Loops - حلقة for

اعمل حاجة على كل عنصر في مجموعة

الـ for loop بتمشي على كل عنصر في مجموعة (list, string, range, dict...) وبتعمل نفس الكود على كل واحد.

الـ range() مهمة جداً مع الـ for:

  • range(5) = من 0 لـ 4 (5 أرقام)
  • range(2, 8) = من 2 لـ 7
  • range(0, 10, 2) = من 0 لـ 9 كل خطوتين (0,2,4,6,8)
python
# على قائمة fruits = ["تفاح", "موز", "كيوي"] for fruit in fruits: print(fruit) # على نص - بيمشي حرف حرف for letter in "معتز": print(letter) # باستخدام range for i in range(5): # 0,1,2,3,4 print(i) for i in range(2, 8): # 2,3,4,5,6,7 print(i) for i in range(0, 10, 2): # 0,2,4,6,8 print(i) # enumerate: احصل على الـ index والقيمة مع بعض for index, fruit in enumerate(fruits): print(f"{index}: {fruit}") # على قاموس person = {"name": "معتز", "age": 25} for key, value in person.items(): print(f"{key}: {value}") # حلقات متداخلة for i in range(1, 4): for j in range(1, 4): print(f"i={i}, j={j}")

Functions - الدوال

كود بتكتبه مرة وتستخدمه أكتر من مرة

الدالة (Function) هي قطعة كود بتديها اسم وتقدر تنادي عليها أي وقت محتاجها. بدل ما تكتب نفس الكود عشر مرات، بتكتبه مرة في دالة وبتنادي على الدالة.

تخيل إنك عندك ماكينة عصير. كل ما تحط فاكهة فيها، بتديك عصير. الدالة زي الماكينة دي - بتديها input (parameters) وبتاخد منها output (return value).

أجزاء الدالة:

  • def: كلمة بايثون لتعريف الدالة
  • اسم الدالة: زي اسم المتغير
  • Parameters: المدخلات بين الأقواس (ممكن يكون فاضية)
  • return: القيمة اللي الدالة بترجعها (اختياري)
  • Default Parameters: قيمة افتراضية لو ما اتديش قيمة
python
# دالة بسيطة def greet(): print("أهلاً يا عالم!") greet() # نادي على الدالة # دالة بـ parameters def greet_person(name): print(f"مرحباً يا {name}!") greet_person("معتز") greet_person("أحمد") # دالة بـ return def multiply(a, b): return a * b result = multiply(4, 7) print(result) # 28 # default parameter def greet_title(name, title="أستاذ"): print(f"أهلاً {title} {name}!") greet_title("محمد") # أهلاً أستاذ محمد! greet_title("عمر", "دكتور") # أهلاً دكتور عمر! # return أكتر من قيمة def calculate(a, b): return a + b, a - b s, d = calculate(10, 3) print(f"الجمع: {s}, الطرح: {d}") # *args: عدد غير محدود من المدخلات def sum_all(*numbers): total = 0 for num in numbers: total += num return total print(sum_all(1, 2, 3)) # 6 print(sum_all(1, 2, 3, 4, 5)) # 15

OOP - البرمجة الكائنية

نظم كودك زي ما الدنيا منظمة

OOP = Object-Oriented Programming. أسلوب في البرمجة بتفكر فيه على إن كل حاجة في العالم هي "كائن" (Object) وكل كائن عنده:

  • Attributes (صفات): زي اسم الشخص وعمره ولونه
  • Methods (أفعال): زي إن الشخص ممكن يتكلم أو يمشي أو ياكل

تخيل إنك بتبني لعبة فيديو. عندك캐ر (Object). الكار عنده: لون، سرعة، عدد العجل (Attributes). والكار يقدر: يتحرك، يوقف، يعدي (Methods). OOP خليك تمثل الكار ده في كود بطريقة طبيعية!

الـ OOP بيحتوي على مبادئ 4:

📦 Encapsulation

تخبي التفاصيل الداخلية وتكشف بس اللي محتاجه. زي الموبايل - مش محتاج تعرف الداخل، بس بتضغط الأزرار.

🧬 Inheritance

Class جديد يورث من class تاني. زي الابن بيورث صفات من أبوه.

🎭 Polymorphism

نفس الاسم بس سلوك مختلف. زي كلمة "يتكلم" - الكلب ينبح والقط يموء.

🔒 Abstraction

إخفاء التعقيد وإظهار الجوهر. زي ما تضغط "طباعة" ومش شايف التفاصيل.

python
# Encapsulation: إخفاء البيانات الحساسة class BankAccount: def __init__(self, balance): self.__balance = balance # __ = private def deposit(self, amount): self.__balance += amount def get_balance(self): return self.__balance account = BankAccount(1000) account.deposit(500) print(account.get_balance()) # 1500 # Inheritance: الوراثة class Animal: def __init__(self, name): self.name = name def speak(self): return "صوت ما" class Dog(Animal): # Dog وارث من Animal def speak(self): # override للميثود return "هاو!" class Cat(Animal): def speak(self): return "مياو!" dog = Dog("ريكو") cat = Cat("كيتي") print(f"{dog.name} بيقول {dog.speak()}") # ريكو بيقول هاو! print(f"{cat.name} بتقول {cat.speak()}") # كيتي بتقول مياو!

Classes & Objects - الكلاسات والأوبجيكتس

عمل الـ Blueprint وصنع الأوبجيكتس منه

الـ Class هو البلانة (Blueprint) - زي رسم الهندسي للبيت. الـ Object هو البيت الفعلي المبني من البلانة دي. من بلانة واحدة ممكن تبني أكتر من بيت!

أجزاء الـ Class:

  • __init__: الدالة الأولى بتتنادي أوتوماتيك لما تعمل object جديد. فيها بتحدد الصفات الابتدائية.
  • self: مرجع للـ object نفسه. أول parameter في أي method في الـ class.
  • Instance Attributes: صفات خاصة بكل object (بتحطها في __init__)
  • Class Attributes: صفات مشتركة بين كل الـ objects
python
class Person: # __init__: بيتنادي أوتوماتيك عند إنشاء object def __init__(self, name, age): self.name = name # instance attribute self.age = age def greet(self): return f"أنا {self.name} عمري {self.age}" def have_birthday(self): self.age += 1 return f"عيد ميلاد سعيد! عمرك دلوقتي {self.age}" # إنشاء objects من الـ class p1 = Person("معتز", 25) p2 = Person("أحمد", 30) print(p1.greet()) # أنا معتز عمري 25 print(p2.greet()) # أنا أحمد عمري 30 print(p1.have_birthday()) # عيد ميلاد سعيد! عمرك دلوقتي 26 # Class Attribute: مشترك بين كل الـ objects class Dog: species = "Canis familiaris" # class attribute def __init__(self, name, breed): self.name = name # instance attribute self.breed = breed d1 = Dog("ريكو", "Golden") d2 = Dog("ماكس", "German") print(d1.species) # نفس للاتنين print(d2.species) # نفس للاتنين # الوراثة مع super() class Student(Person): # وارث من Person def __init__(self, name, age, student_id): super().__init__(name, age) # نادي على init الأب self.student_id = student_id def study(self): return f"{self.name} بيذاكر" s = Student("سارة", 20, "S001") print(s.greet()) # وارث من Person print(s.study()) # خاص بـ Student

Read Files - قراءة الملفات

طرق مختلفة لقراءة محتوى الملف

في عدة طرق لقراءة محتوى ملف، كل واحدة ليها استخدامها:

  • read(): اقرأ كل الملف مرة واحدة كنص واحد
  • read(n): اقرأ أول n حرف
  • readline(): اقرأ سطر واحد بس
  • readlines(): اقرأ كل الأسطر وحطهم في قائمة
  • for line in file: امشي على الأسطر واحد واحد (أكفأ للملفات الكبيرة)
python
# قراءة كل الملف with open("example.txt", "r") as file: content = file.read() print(content) # قراءة أول 10 حروف with open("example.txt", "r") as file: first_ten = file.read(10) print(first_ten) # قراءة سطر سطر with open("example.txt", "r") as file: line1 = file.readline() line2 = file.readline() print("السطر الأول:", line1.strip()) print("السطر التاني:", line2.strip()) # كل الأسطر في قائمة with open("example.txt", "r") as file: lines = file.readlines() print(lines) # ['سطر 1\n', 'سطر 2\n', ...] # المشي على الأسطر في حلقة (الأكفأ) with open("example.txt", "r") as file: for line in file: print(line.strip()) # strip بيشيل الـ \n # قراءة مع معالجة الخطأ try: with open("مش_موجود.txt", "r") as file: content = file.read() except FileNotFoundError: print("الملف مش موجود!")

كتابة بيانات متنوعة وحذف الملفات بأمان

قدرنا نكتب نصوص بسيطة في الملفات، بس ممكن نكتب قوائم وقواميس كمان. وأهم حاجة هي الحذف الآمن باستخدام مكتبة os.

دايماً تأكد إن الملف موجود قبل ما تحاول تحذفه! لو الملف مش موجود وحاولت تحذفه، هتاخد FileNotFoundError.
python
import os # كتابة قاموس في ملف student = {"name": "معتز", "age": 25, "grade": "A"} with open("student.txt", "w") as file: for key, value in student.items(): file.write(f"{key}: {value}\n") # حذف ملف واحد بأمان if os.path.exists("student.txt"): os.remove("student.txt") print("الملف اتحذف!") else: print("الملف مش موجود أصلاً") # حذف أكتر من ملف files_to_delete = ["ملف1.txt", "ملف2.txt", "ملف3.txt"] for filename in files_to_delete: if os.path.exists(filename): os.remove(filename) print(f"اتحذف: {filename}") # حذف مع معالجة كاملة للأخطاء try: os.remove("ملف_مهم.txt") print("اتحذف بنجاح") except FileNotFoundError: print("الملف مش موجود") except PermissionError: print("مش عندك صلاحية الحذف") except Exception as e: print(f"خطأ: {e}")
